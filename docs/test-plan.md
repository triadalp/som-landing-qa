# Test Plan

## 1. Project Information

**Project Name:** Serwis S.O.M. Landing Page

**Project Type:** Marketing Landing Page

**Domain:** Automotive Service

**Application Type:** Web Application

**Testing Type:** Manual Testing

---

## 2. Objective

The objective of testing is to verify that the landing page functions correctly, provides a positive user experience and allows users to successfully contact the auto service through available communication channels.

Special attention is given to the contact form, as it is the primary lead generation feature of the application.

---

## 3. Scope

### In Scope

- Header and navigation
- Hero section
- Services section
- About section
- Reviews section
- Contact section
- Contact form validation
- Contact links
- Responsive layout
- Basic UI verification

### Out of Scope

- Backend logic
- API testing
- Database testing
- Performance testing
- Security testing
- Cross-platform mobile application testing

---

## 4. Test Strategy

Testing will be performed manually from the end-user perspective.

The testing process includes:

- Functional testing
- UI testing
- Responsive testing
- Positive testing
- Negative testing
- Retesting after bug fixes
- Basic regression testing of related functionality

---

## 5. Test Environment

Operating System:
- Windows 11

Browsers:
- Google Chrome
- Mozilla Firefox
- Microsoft Edge

Devices:
- Desktop

- Tablet viewport

Tools:
- Chrome DevTools
- Git
- GitHub

---

## 6. Entry Criteria

Testing starts when:

- The application is deployed and accessible.
- Core functionality is implemented.
- The tester has access to the application.

---

## 7. Exit Criteria

Testing is considered complete when:

- All planned checklist items are executed.
- Critical functionality has been verified.
- All identified defects are documented.
- Retesting of fixed defects has been completed.
- Test Summary Report has been prepared.

---

## 8. Risks

Potential risks include:

- Incorrect contact form validation.
- Broken contact links.
- Responsive layout issues.
- UI inconsistencies on different screen sizes.

---

## 9. Deliverables

The following testing documentation is prepared:

- Product Analysis
- Test Scope

- Bug Reports
